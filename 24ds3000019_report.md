## 1\. Introduction

Chest X-ray imaging is one of the most widely used diagnostic tools in clinical practice for identifying diseases in the thoracic region. Manual analysis by radiologists is time-consuming and prone to inconsistency. This project aims to automate that process using deep learning.

The dataset contains 51,043 labeled training images and 17,015 unlabeled test images. Each image belongs to exactly one of 20 thoracic pathology classes including Atelectasis, Pneumonia, Effusion, and No Finding. The task is a multi-class classification problem formulated with an asymmetric cost function that penalizes False Negatives five times more heavily than False Positives, reflecting the clinical reality that missing a disease is far more dangerous than raising a false alarm.

The objective was to build a model that correctly identifies the pathology class of each chest X-ray image, with a particular focus on minimizing missed disease detections.

## 2\. Methodology

### Model Selection

EfficientNetB0 pretrained on ImageNet was chosen as the backbone for transfer learning. The key reason for this choice was the extreme class imbalance in the dataset. The rarest class, Pneumomediastinum, had only 5 training samples. Training a model from scratch on 5 samples is not feasible. EfficientNetB0 already knows how to detect edges, textures, and structural patterns from 1.2 million ImageNet images. Fine-tuning its final layers for chest X-ray classification was far more practical than building a custom CNN from scratch.

EfficientNetB0 was also chosen over larger models like ResNet50 or DenseNet121 because it offers a better balance of speed and accuracy within Kaggle's GPU time limits.

The final architecture was:

- EfficientNetB0 base (frozen, pretrained on ImageNet)  
- GlobalAveragePooling2D  
- Dense(128, activation=relu)  
- Dropout(0.3)  
- Dense(20, activation=softmax)

### Preprocessing

- Images were already uniformly sized at 384x384 pixels with 3 RGB channels, confirmed during EDA  
- Resized to 224x224 to match EfficientNetB0 input requirements  
- Pixel values normalized to \[0, 1\] by dividing by 255  
- One-hot encoded labels converted to integer class indices for training  
- Training set split into 80% train and 20% validation using stratified splitting to preserve class distribution

### Data Augmentation

Light augmentation was applied to the training set only:

- Horizontal flip  
- Rotation up to 10 degrees  
- Zoom up to 10%  
- Width and height shift up to 5%

Aggressive augmentation was avoided since X-ray orientation and structure carry clinical meaning.

### Handling Class Imbalance

Class imbalance was a major challenge with an imbalance ratio of 6815 between the most and least frequent classes. Two approaches were explored across experiments, detailed in the next section.

### Loss Function

Categorical cross-entropy was used as the loss function. This is appropriate for multi-class single-label classification where the model outputs a probability distribution across 20 classes via softmax.

## 3\. Experiments and Results

Three experiments were performed, each improving on the previous based on observed issues.

### Experiment 1: Initial Run with Class Weights

**Configuration:**

- Learning rate: 0.001  
- Class weights: balanced (computed using sk-learn, capped at 20\)  
- class\_mode: categorical  
- Epochs: up to 15 with early stopping (patience=3)

**Issue Discovered:**  
During this run, a critical bug was found in the label-to-generator mapping. Keras sorts string labels alphabetically, so label strings '0' through '19' were sorted as '0', '1', '10', '11' instead of numerically. This caused completely wrong class assignments. The model predicted Calcification of the Aorta (the class mapped to index 18 due to wrong sorting) for every single test image.

**Result:** \-34.0

### Experiment 2: Bug Fixed, Class Weights Applied

**Configuration:**

- Label strings zero-padded to '00' through '19' to fix alphabetical sorting  
- Learning rate: 0.001  
- Class weights capped at 20.0  
- Epochs: 5 (early stopping triggered)

**Observations:**  
Training was unstable. Validation accuracy swung between 0.003 and 0.667 across epochs because the high class weights for rare classes (Pneumomediastinum had weight 510 before capping) caused the optimizer to take erratic steps.

| Epoch | Train Acc | Val Acc | Val Loss |
| :---- | :---- | :---- | :---- |
| 1 | 0.1063 | 0.0031 | 2.841 |
| 2 | 0.0196 | 0.6676 | 2.785 |
| 3 | 0.1790 | 0.0076 | 2.821 |

**Result:** \-5.80

### Experiment 3: Class Weights Removed, Lower Learning Rate (Final Submission)

**Configuration:**

- Learning rate: 0.0001  
- No class weights  
- Early stopping patience increased to 5  
- Epochs: 14 (early stopping triggered, best epoch was epoch 9\)

**Observations:**  
Training became stable. Loss decreased steadily across all epochs with no swinging behavior.

| 9 | 0.6691 | 0.6676 | 1.3787 | Yes (best) |

Best model restored from epoch 9\.

**Result:** \-4.72537

## 4\. Conclusion

This project implemented an automated chest X-ray classification system using transfer learning with EfficientNetB0 across 20 thoracic pathology classes. The final Kaggle score achieved was \-4.72537.

The most critical learning from this project was the importance of verifying data pipeline correctness before training. A subtle bug in string label sorting caused completely wrong class mappings, which produced a score of \-34 despite the model architecture being correct. Fixing the data pipeline alone improved the score from \-34 to \-5.80.

The second key learning was that class weights, while theoretically important for imbalanced datasets, need to be applied carefully. Extreme weights for very rare classes destabilized training and produced worse results than training without weights at a lower learning rate.

The final model predicts the majority class (No Finding) for most images, which is expected given that No Finding comprises 67% of training data and no class weighting was applied. Future improvements would include fine-tuning the top layers of EfficientNetB0, applying mild class weights capped at a conservative value, and using a larger input resolution of 260x260.
